# /til-plan — 블로그 기획서 생성

주제: **$ARGUMENTS**

## 🔍 철저성 원칙 (Thoroughness Mandate)

이 커맨드의 모든 단계는 **상세히, 꼼꼼히, 자세히** 수행합니다. 대충 훑는 조사는 허용되지 않습니다.

- 에이전트 조사 결과를 하나도 빠뜨리지 말고 기획서에 반영
- 목차는 추측이 아니라 실제 조사 결과에 기반해 구성
- 참고 자료는 권위 있는 출처의 직접 URL로 자세히 기재
- 기획서 완성 전 self-review로 누락·모순 항목이 없는지 꼼꼼히 점검 (최소 2회)

## 절차

### 1. 배치를 자세히 결정
- Glob/Grep으로 `docs/` 하위 전체를 상세히 스캔해 유사 주제 문서 존재 여부 확인
- 적절한 섹션(`frontend` / `algorithm` / `tools`)과 하위 카테고리 폴더를 꼼꼼히 선정
- 해당 폴더 내 파일들의 숫자 prefix를 모두 확인해 최대값 + 1로 새 파일번호 결정
- 기존 유사 문서가 있으면 "새로 작성 vs 보완" 옵션을 함께 제시

### 2. 에이전트 병렬 조사 (상세한 결과 수집)
한 메시지에서 Agent tool을 다중 호출하여 병렬 실행:

- **frontend-skill-explorer** — 주제의 하위 개념, 선행지식, 관련 기술, 흔한 오해를 상세히 조사
- **frontend-blog-verifier** — 권위 있는 참고자료 URL, 팩트체크 필요 주장, 버전별 차이를 자세히 파악
- **frontend-ui-example-generator** — UI/시각화가 필요한 주제일 때만. 예제 아이디어 3~5개를 꼼꼼히 제안

각 에이전트에게 "조사만 수행하고 본문은 쓰지 말 것. 빠뜨리지 말고 상세히 제공할 것." 명시적 지시.
순수 개념·알고리즘 포스트면 UI 에이전트 생략.

### 3. 기획서를 꼼꼼히 작성 후 저장

경로: `plan/<주제-영문-슬러그>.md` (한국어 주제면 영문 슬러그로 변환. 예: "HTML h태그" → `plan/html-h-tag.md`)
`plan/` 폴더가 없으면 Write tool이 자동 생성.

기획서 구조(**모든 섹션을 상세히 채울 것, 빈 섹션·"TBD" 금지**):

```markdown
# 📝 블로그 기획서: <주제>

## 📍 배치
- 대상 경로: `docs/<section>/<subcategory>/<NN>-<filename>.md`
- 유사 기존 문서: <경로 또는 "없음">

## 🎯 대상 독자
- 수준: 초급/중급/고급
- 선행 지식: (bullet로 꼼꼼히 열거)

## 💡 핵심 메시지
<독자가 읽고 얻어갈 한 줄 — 모호하지 않게 자세히>

## 📑 목차 초안
(각 섹션을 한 줄 설명과 함께 상세히)
1. 개념 — ...
2. 동작 원리 — ...
3. 예제 — ...
4. 주의사항 — ...
5. 정리 — ...

## 🔍 섹션별 상세
### 1. <섹션명>
- 다룰 포인트: (bullet로 자세히)
- 코드 예제: <필요 여부 + 어떤 예제>
- 시각 자료: <필요 여부>
...

## 📚 참고 자료 (권위 있는 출처 — URL 포함)
- <URL>

## ⚠️ 작성 시 검증 필요 항목
- (팩트체크 대상 주장을 빠뜨리지 말고 상세히 나열)

## 🎨 예제/시각 자료 후보 (해당 시)
- <UI 에이전트 제안을 꼼꼼히 반영>

## 🚫 이번 포스트 범위 밖
- <스코프 밖 주제를 명시적으로 나열 — scope creep 방지>

## ➡️ 다음 단계
`/til-write plan/<파일명>.md` 로 본문 작성 단계 진행.
```

### 4. 결과를 상세히 알림
- 저장된 기획서 경로
- 다음 단계 명령어(`/til-write ...`) 안내
- 기획서에서 특히 주의 깊게 검토해야 할 섹션(⚠️ 검증 필요 항목 등) 강조

## 제약
- 본문은 쓰지 않음 (기획까지만, 엄격히)
- 한국어 콘텐츠, 코드·기술 용어는 영어
- 목차가 7개 이상이면 2~3개 포스트로 분할 제안을 기획서에 자세히 명시
- 에이전트 결과의 불확실한 부분은 "⚠️ 검증 필요" 항목으로 명시 이관
- 기획서 저장 후 Read로 재확인해 의도대로 저장됐는지 꼼꼼히 검증
