# CLAUDE.md

이 파일은 이 저장소에서 작업하는 Claude Code(claude.ai/code) 인스턴스에게 가이드를 제공합니다.

## 프로젝트 개요

**Docusaurus 3** 기반의 개인용 한국어 TIL("Today I Learned") 사이트로, GitHub Pages(https://jjamva.github.io/FE-TIL-JJamVa/)에 배포됩니다. Front-End, Algorithm, Tools 세 가지 주제를 다룹니다. 콘텐츠 중심 프로젝트이므로, 대부분의 커밋은 코드가 아니라 `docs/` 하위의 Markdown 파일을 추가하거나 수정하는 작업입니다.

## 명령어

패키지 매니저는 **pnpm**입니다 (`pnpm-lock.yaml` 참조). Node 버전은 `>=16.14`.

- `pnpm start` — 핫 리로드가 적용된 로컬 개발 서버 실행
- `pnpm build` — `build/` 폴더로 정적 빌드 생성
- `pnpm serve` — 빌드된 사이트를 로컬에서 서빙
- `pnpm clear` — Docusaurus 캐시(`.docusaurus/`) 삭제. 사이드바나 config 변경이 반영되지 않을 때 실행
- `pnpm deploy` — `gh-pages` 브랜치로 배포 (실행 전에 `GIT_USER` 환경 변수 설정 필요)

테스트, 린터, 타입체크 스크립트는 설정되어 있지 않습니다.

## 콘텐츠 구조

`docs/` 하위의 3개 최상위 섹션은 각각 네비게이션 탭으로 노출되며, **autogenerated sidebar**(`sidebars.js`)를 통해 자동 렌더링됩니다:

- `docs/frontend/` — HTML, CSS, JavaScript, jQuery, Tailwind, Bootstrap, Sass, React, TypeScript, Next.js, React Native
- `docs/algorithm/` — BFS, DFS, DP
- `docs/tools/` — Git, VSCode, Figma

각 섹션에는 `0-intro.md` 진입 파일이 있습니다. 하위 카테고리는 숫자 prefix가 붙은 폴더(예: `1-HTML/`, `3-JavaScript/`)이며, 내부 파일도 동일한 네이밍 규칙을 따릅니다(`0-HTML 개념.md`, `1-목록 관련 태그.md` 등). **숫자 prefix가 사이드바 정렬 순서를 결정**하므로, 새 파일을 추가할 때는 기존 순서를 유지하고 폴더 번호 체계를 따라야 합니다.

`docusaurus.config.js`의 navbar는 각 섹션의 `{section}/intro` doc ID를 참조합니다. `intro.md` 파일을 이름 변경하거나 삭제할 경우 navbar의 `docId` 필드도 함께 업데이트해야 합니다.

콘텐츠 언어는 한국어이며, 코드 블록과 기술 용어는 영어 그대로 유지합니다.

## 코드 영역

최소한의 코드만 존재 — 앱이 아닌 문서 사이트입니다:
- `src/pages/index.tsx` — 랜딩 페이지
- `src/components/Comments.tsx` — 유일한 커스텀 컴포넌트
- `src/theme/` — Docusaurus swizzle로 오버라이드한 테마
- `src/css/custom.css` — 전역 스타일
- `static/img/` — 문서에서 참조하는 이미지

TypeScript는 `@tsconfig/docusaurus`를 확장 — 커스텀 빌드 파이프라인 없음.

## 배포 설정

`docusaurus.config.js`의 주요 설정: `organizationName: jjamva`, `projectName: FE-TIL-JJamVa`, `deploymentBranch: gh-pages`, `baseUrl: /FE-TIL-JJamVa/`. `trailingSlash: true`가 설정되어 있으므로, 내부 링크도 이 규칙에 맞춰야 합니다.
